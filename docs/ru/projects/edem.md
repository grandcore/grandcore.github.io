## Цель проекта

Создать максимально универсальный и лёгкий для кастомизации инструмент, который позволит автоматизировать бизнес-процессы любой сложности, как в открытых демократических сообществах, так и в вертикальных организациях с чётким разграничением прав у разных групп пользователей.

## Краткое описание

?> Разработка экспериментальной версии ведётся в рамках проекта [OpenSchool](ru/projects/openschool.md), после чего данный сервис будет выделен в отдельный проект.

**Общая концепция**

EDem встраивается как отдельный сервис в существующую инфраструктуру. Он принимает запрос на создание процесса и входные данные, система, опираясь на соответствующий скрипт процесса, регистрирует его в системе, выбирает участников и устанавливает им роли, а также выполняет другие действия, например, устанавливает отслеживание событий.

**Пользовательский опыт**

![Пример процесса](../../_media/openschool-4-6.png)

Каждый пользователь находится в группах, которые имеют доступ к определённым типам процессов и роли в их рамках. Когда процесс регистрируется - участники получают уведомление, а процесс отмечается в их списке как необработанный. Интерфейс окна взаимодействия пользователя с процессами состоит из трёх основных блоков.

Творческий блок - занимает центральное место. В нём происходит работа над определенными задачами, интерфейс отрисовывается выбранным движком процесса. Система имеет движок по умолчанию, который может выводить данные из входного узла процесса и позволяет заполнять формы для передачи в выходной узел процесса после его принятия. Также, в этом блоке может быть, например, интерфейс для работы с кодом, графикой, чертежами, обработки обращений по различным каналам связи, в том числе, работы совместной, в реальном времени.

Чат процесса - даёт возможность в реальном времени общаться группам пользователей имеющих к нему доступ.

Блок принятия предназначен для отправки данных в выходной узел. В самом простом случае это может быть кнопка "применить", которую нажимает участник имеющий на это право. При этом, в данном блоке может быть несколько вариантов, которые отправят принятые данные по другому маршруту. Это может быть, например, голосование или обязательное подтверждение нескольких участников.

Права участников процесса гибко настраиваются на все три блока для разных групп пользователей. Например, у обсуждения может быть отключена возможность писать или даже видеть этот блок. У блока принятия решений может быть недоступна статистическая информация или он также может быть скрыт. Движку творческого блока отправляется код доступа для каждой отдельной группы пользователей для самостоятельного распределения прав. Участники с необходимыми правами могут добавлять к процессу новых участников, давая им определённые права доступа.

**Особенности**

Обычно в таких системах алгоритмы процессов описываются с помощью визуального программирования. В EDem же используется язык JavaScript. Это даёт максимальную гибкость, при этом, не сильно усложняет, если не упрощает настройку.

Интерфейс взаимодейсвия пользователя с процессами нацелен на работу в потоке. По завершении работы с одним процессом, показывается следующий по приоритету процесс, без необходимости выходить в общий список. Приоритетность у процессов реализована как для всех пользователей, так и для каждого в отдельности. В первую очередь выводятся карточки имеющие личный приоритет начиная от наивысшего, после чего - процессы с общим приоритетом. После завершения процессов с приоритетом выводятся остальные, начиная с самого старого. Некоторые процессы могут иметь спецприоритет, те карточка показываются принудительно, даже если пользователь обрабатывает в этот момент другойй процесс. Это может быть полезно, например, для принятия телефонного звонка. 

Каждый процесс может отслеживать внешние и внутренние события, выполняя определённые скрипты, например, принудительно завершать процесс или открывать новый, а также отправлять запросы в другие процессы или внешние сервисы. После завершения процесса возможно не только передать данные во внешние сервисы, но и открыть новый процесс на основе обработанных данных. 

## Примеры использования

- **Совместная работа над проектами:** написание текстов и их перевод, производство видео и аудио, систематизация контента...

**Саморегуляция политических и общественных организаций:** принятие внутренних документов, голосование за предложения участников, назначение на должности...

**Автоматизация работы организации:** воронки для обработки обращений, совместная работа и контроль выполнения задач...
