# Философия фонда

!> Документ находится в разработке. Если у вас есть вопросы или предложения, пишите - [@grandcore](https://t.me/grandcore).

## Манифест

- Основная цель фонда - сформировать сообщество и создать платформу для эффективной работы над свободными проектами: свободного ПО, этичных онлайн-сервисов, стандартов и чертежей для различных изделий.
- Всё созданное в рамках фонда становится общественным достоянием. Любой может использовать наработки фонда по своему усмотрению, в том числе, создавая коммерческие реализации на их основе.
- Фонд управляется сообществом с помощью демократических процессов. Вес голоса участника пропорционален его компетенциям, материальному и созидательному вкладу. Принятием ключевых решений и разрешением споров путём равноправного голосования занимаются Контрибьюторы-Хранители, состоящие из сооснователей фонда и участников внёсших особый вклад в его развитие. Наделение статусом нового Хранителя или лишение данного статуса существующего Хранителя происходит общим голосованием участников сообщества наделённых данным статусом путём равного голосования. Основатель фонда не имеет никаких дополнительных привилегий перед другими Хранителями.
- Участники сообщества могут зарабатывать деньги выполняя задачи для открытых проектов на которые собираются средства через встроенный сервис краудфандинга, продавать дополнения для ПО и изделия сделанные по открытым стандартам и чертежам фонда в маркетплейсе, выполнять заказы по внедрению и доработкам проектов под конкретные нужды на бирже услуг.
- Фонд зарабатывает с каждой транзакции проведённой на платформе. Как при сборах средств на выполнение задач у проектов, так и с продаж в маркетплейсе или услугах гаранта на бирже услуг. В крайнем случае - будет открываться прямой сбор на нужды фонда. При этом, все деньги идут только на поддержание работы фонда и его проектов. Ни один из участников фонда, в том числе, основатель - не может получать какие-либо дивиденды.

## Дополнительные пояснения

- Разработка всех проектов будет регламентирована едиными стандартами и практиками.
- В архитектуру всех социальных проектов фонда будет закладываться чёткое разделение на стандарты информации и её передачи, сервисы её предоставляющие и клиентский софт для её потребления. То есть, пользователь сможет выбрать интересные ему независимые друг от друга каналы предоставления информации, клиент для потребления контента, сервис рекомендации, сообщества для его обсуждения и так далее.
